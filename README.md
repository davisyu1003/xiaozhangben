# 小鱼儿账本 - 云端配置指南

## 功能更新说明

本次更新实现了以下两个主要功能：

1. **云端数据存储** - 使用 jsonbin.io 保存账本数据，多设备同步
2. **API Key 云端配置** - 避免代码中明文展示敏感信息

---

## 配置步骤

### 1. 创建数据存储 Bin

1. 访问 https://jsonbin.io/
2. 注册/登录账号
3. 创建一个新的 Bin（用于存储账本数据）
4. 记录 Bin ID（如：`67df7c8eacd3cb34a8f0f3a0`）
5. 在 API Keys 页面生成：
   - **Read Access Key**（只读密钥）- 用于获取数据
   - **Write Access Key**（写入密钥）- 用于保存数据

### 2. 创建配置 Bin（用于存储 API Key）

1. 在 jsonbin.io 创建另一个 Bin
2. 添加以下内容：

```json
{
  "apiKey": "你的 OpenRouter API Key"
}
```

3. 记录这个 Bin ID（用于 `CONFIG_URL`）
4. 设置为 **Public** 或使用只读密钥

### 3. 修改代码中的配置

打开 `index.html`，找到配置区域：

```javascript
// ==================== 配置区域 ====================
const CONFIG_URL = "https://api.jsonbin.io/v3/b/你的配置BIN_ID/latest";

// 数据存储配置
const DATA_BIN_ID = "你的数据BIN_ID";
const DATA_READ_KEY = "$2a$10$你的只读密钥";
const DATA_WRITE_KEY = "$2a$10$你的写入密钥";
// =================================================
```

替换为实际的值。

## 工作原理

### 数据同步流程

1. **首次加载**：
   - 尝试从云端获取数据
   - 如果云端有数据，使用云端数据
   - 如果云端为空，使用本地数据并上传
   - 如果云端连接失败，使用本地缓存

2. **数据变更**：
   - 每次添加/修改/删除记录后
   - 自动同步到云端（1秒防抖）
   - 同时更新本地缓存

3. **多设备同步**：
   - 设备 A 添加记录 → 自动同步到云端
   - 设备 B 打开页面 → 从云端获取最新数据
   - 设置页面可手动触发同步

### API Key 安全

- API Key 存储在独立的配置 Bin 中
- 代码中不再包含明文 API Key
- 配置 Bin 可以设置为只读权限
- 需要时从云端动态获取

---

## 注意事项

1. **jsonbin.io 免费版限制**：
   - 每日请求次数有限
   - 存储空间有限
   - 如需更多资源，可考虑升级或使用其他服务

2. **数据安全**：
   - Write Key 具有完全访问权限，请妥善保管
   - 不要在公开场合分享包含 Write Key 的代码

3. **离线使用**：
   - 应用支持离线使用
   - 离线时的数据变更会在下次联网时同步

---

## 替代方案

如果不想使用 jsonbin.io，可以考虑：

1. **Firebase Firestore** - Google 提供的实时数据库
2. **Supabase** - PostgreSQL 后端即服务
3. **LeanCloud** - 国内云服务，访问速度快
4. **自建后端** - 使用 Node.js + MongoDB 等

这些方案需要相应的代码调整。
